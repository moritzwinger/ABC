void SpecialTypeCheckingVisitor::discardChildrenDatatypes ( )

◆ registerBeforeVisitChildren()

void SpecialTypeCheckingVisitor::registerBeforeVisitChildren ( )

Remembers the number of nodes in typesVisitedNodes. After the children have been visited, calling discardChildrenDatatypes removes the datatype of the visited children from the stack. This mechanism is required for nodes that do not require children's type information or where the number of children is arbitrary and hence a simple pop() after visiting the children is not sufficient.
