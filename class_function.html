Generically replaces a child node with a different node. This is computationally inefficient, iterating through all children and searching for a match It also requires a lot of dynamic casting, therefore: If it's possible, it is preferred to replace children directly through the derived class interface (e.g. setX)

Parameters
childReference to the child that needs to be replaced (compared by address!)
new_childNew node to insert, must have suitable type!
Returns
Ownership of the old child
Exceptions
